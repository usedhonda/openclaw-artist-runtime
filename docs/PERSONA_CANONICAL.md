# Persona Canonical Contract

Artist Runtime keeps persona input small and keeps generated files managed. This
is the source of truth for the five top-level persona Markdown files and the
runtime current-state projection.

## Ownership Map

| File or setting | Class | User editable | Required for setup | Source of truth | Runtime behavior |
| --- | --- | --- | --- | --- | --- |
| `artist.identity.displayName` | Config identity | Yes, in Settings | Optional | `runtime/config-overrides.json` | Supplies the display name used by generated identity projection. |
| `artist.identity.producerCallname` | Config identity | Yes, in Settings | Optional | `runtime/config-overrides.json` | Supplies the producer callname used by generated identity projection. |
| `ARTIST.md` | Canonical user input | Yes | Required | Managed `artist-runtime:persona:core` block plus preserved creative notes | Owns artist concept, sound, obsessions, lyrics stance, public voice, and Suno production traits. It must not own artist display name or producer facts. |
| `SOUL.md` | Canonical user input | Yes | Required | Managed `artist-runtime:persona:soul` block | Owns conversation voice, tone, refusal style, and speaking moves. It must not own producer callname or music production rules. |
| `PRODUCER.md` | Canonical user input | Yes | Optional | Managed `artist-runtime:persona:producer` block | Owns producer facts and preferences that change creative decisions. It must not contain secrets. |
| `IDENTITY.md` | Derived projection | No | No | Generated from config identity plus `ARTIST.md` and `SOUL.md` | Runtime writes and refreshes the physical file before API display and prompt consumers read it. Hand-written stale content is archived once under `runtime/persona-legacy/` and overwritten. |
| `INNER.md` | Internal runtime state | No | No | Runtime-managed file | Prompt consumers may read it, but setup and persona API writes reject direct edits. |
| `artist/CURRENT_STATE.md` | Runtime-managed projection | No | No | Runtime-managed file | Tracks current creative pull. Blank or old placeholder contents are replaced with safe managed defaults before prompt reads. |
| `artist/PRODUCER_NOTES.md` | Producer notes | Yes, outside setup | No | Operator-authored workspace note | Holds producer notes that do not belong in persona canonical fields. |

## Workspace Boundary

The default live workspace is `.local/openclaw/workspace`. Repository-root
persona files are package/reference artifacts unless an operator explicitly sets
`artist.workspaceRoot` to another path.

This keeps local runtime history, generated projections, Suno artifacts, and
private producer material out of the distributable repository root.

## Write Rules

- Setup and Producer Console persona editing write only `ARTIST.md`, `SOUL.md`,
  and `PRODUCER.md`.
- `IDENTITY.md` is regenerated by the identity projection service after
  canonical/config changes and before prompt consumers read it.
- `INNER.md` is not a setup target and is not accepted by the persona patch API.
- `artist/CURRENT_STATE.md` is not a setup target. It is runtime-managed and
  sanitized before use.
- Cleanup may archive and remove stale owner fields outside the managed
  `ARTIST.md` marker block, such as old artist-name or producer-callname lines.
  Creative text outside the marker block is preserved.
